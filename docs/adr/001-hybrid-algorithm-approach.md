# ADR-001：选择双引擎混合算法方案

- **状态（Status）**：已接受（Accepted）
- **日期**：2026-06-22
- **决策者**：第 12 组（算法负责人 黄权达、算法副 张嘉豪、组长 陈绍杰）

## 背景（Context）

系统需要同时完成两件事：**品质分级**（一级 / 二级 / 等外品）与**表面缺陷检测**（病斑 / 裂纹 / 碰伤 / 畸形）。面临的约束：

- 课程为软件工程方法论课，算法只需"能演示、有结果"，实现仅占 14% 评分。
- 数据量有限（演示级 300~600 张），深度学习模型精度难保证。
- 演示环境可能无 GPU，需 CPU 可推理。
- 分级结论需**可解释**（不能黑盒给等级），便于答辩讲解。

可选方案：
1. 纯深度学习（端到端分类 + 检测）——精度依赖数据，黑盒，风险高。
2. 纯传统 CV——缺陷检测能力弱，难框选定位。
3. 混合：传统 CV 做分级 + 深度学习做缺陷检测。

## 决策（Decision）

采用**双引擎混合方案**：

- **缺陷检测**用深度学习 **YOLOv8n/s**：擅长目标定位与分类，输出缺陷框 + 类别 + 置信度。
- **品质分级**用**传统 CV + 规则引擎**：基于果径、着色率、缺陷占比量化指标，按可配置阈值判级，结论可解释。
- 两引擎结果融合：缺陷占比由检测结果计算，反馈到分级规则。

## 后果（Consequences）

**正面**：
- 分级可解释，附量化依据，答辩易讲。
- 传统 CV 分级链路独立，DL 不可用时可兜底（见 [ADR-002]）。
- YOLOv8n CPU 可推理，适配无 GPU 演示。
- 风险分散：分级不依赖模型精度。

**负面 / 代价**：
- 需维护两套逻辑（CV pipeline + DL 模型）。
- 分级阈值需人工调优与标注一致性验证（目标一致率 ≥ 80%）。

**关联**：分级阈值落地于 `grade_standard` 表，可由管理员在线配置；详见详细设计 §6。
