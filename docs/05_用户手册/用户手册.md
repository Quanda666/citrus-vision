## 1. 系统简介

CitrusVision 是一套面向肇庆柑橘（砂糖橘 / 贡柑）的机器视觉品质检测系统。上传柑橘图片，系统自动给出品质等级（一级 / 二级 / 等外品）、表面缺陷框选（病斑 / 裂纹 / 碰伤 / 畸形）与量化指标（果径 / 着色率 / 缺陷占比），并支持溯源登记、价格参考与报表导出。

适用人员：

- **普通用户**：分拣员、质检员 —— 日常检测、查看与导出。
- **管理员**：技术员、管理人员 —— 在普通用户能力之上维护分级标准、价格、模型与日志。

## 2. 安装部署

系统由四个服务组成：前端（:80）、后端 API（:8000）、算法服务（:8001）、MySQL（:3306）。提供两种部署方式。

### 2.1 方式一：Docker Compose 一键启动（推荐）

前置：已安装 Docker 与 Docker Compose。

```bash
# 1. 进入项目根目录
cd citrus-vision

# 2. 启动全部服务（首次会构建镜像）
docker-compose up -d

# 3. 查看状态
docker-compose ps

# 4. 访问
# 浏览器打开 http://localhost
```

服务编排见 [docker-compose.yml](../../docker-compose.yml)。MySQL 首次启动会自动执行 `database/schema.sql` 与 `seed.sql` 初始化。

> **待补文件提示**：当前三个服务的 `Dockerfile` 与 `deploy/nginx.conf` 尚未创建，需先补齐再执行构建。
> - `frontend/Dockerfile`：基于 node 构建 + nginx 托管，拷贝 `deploy/nginx.conf`
> - `backend/Dockerfile`：基于 python:3.10，装 requirements，`uvicorn main:app --host 0.0.0.0 --port 8000`
> - `algorithm-service/Dockerfile`：基于 python:3.10，装 requirements，启动 `:8001`
> - `deploy/nginx.conf`：托管前端静态文件，`/api` 反向代理到后端 `:8000`

### 2.2 方式二：本地手动启动

前置：Python 3.10+、Node.js 18+、MySQL 8.0+。

```bash
# 1. 数据库初始化
mysql -u root -p < database/schema.sql
mysql -u root -p < database/seed.sql
```

> ⚠️ **建表顺序提示**：`detection_task` 外键引用 `model_info`，若直接执行报外键错误，请先在 schema.sql 顶部加 `SET FOREIGN_KEY_CHECKS=0;`、底部加 `SET FOREIGN_KEY_CHECKS=1;`，或调整建表顺序使 `model_info` 先建。

```bash
# 2. 后端
cd backend
python -m venv venv && source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
uvicorn main:app --reload --port 8000

# 3. 算法服务（另开终端）
cd algorithm-service
pip install -r requirements.txt
python server.py        # 监听 :8001

# 4. 前端（另开终端）
cd frontend
npm install
npm run dev             # 开发模式，默认 :5173
```

也可使用一键脚本 [deploy/run_local.sh](../../deploy/run_local.sh) 启动、[deploy/stop_local.sh](../../deploy/stop_local.sh) 停止。

## 3. 普通用户操作指南

### 3.1 登录

1. 打开系统首页，进入登录页。
2. 输入用户名、密码，点击"登录"。
3. 登录成功进入仪表盘（Dashboard）。

### 3.2 单张检测（核心）

1. 进入"检测"页面（Detect）。
2. 点击或拖拽上传一张柑橘图片（JPG/PNG，≤10MB）。
3. 点击"开始检测"，等待结果（CPU 约 ≤3 秒）。
4. 查看结果：
   - **标注图**：缺陷以彩色框标出，标注类别与置信度。
   - **等级徽章**：一级（精品）/ 二级（合格）/ 等外品。
   - **量化指标卡**：果径（mm）、着色率（%）、缺陷占比（%）。
   - **缺陷列表**：每个缺陷的类型与置信度。
   - **参考价格**：按等级显示市场参考价（元/斤）。

### 3.3 批量检测

1. 进入"批量检测"页（Batch）。
2. 一次选择多张图片（≥10 张）上传。
3. 点击"开始检测"，查看进度条。
4. 完成后查看结果列表与统计汇总（各等级数量、缺陷分布）。

### 3.4 查看历史任务

进入"任务列表"（TaskList），按状态 / 时间筛选，点击任务查看详情结果。

### 3.5 溯源登记

1. 在结果页点击"溯源登记"。
2. 填写产地、品种、生产者、批次、检测人。
3. 保存后系统生成溯源二维码，扫码可查看溯源信息。

### 3.6 报表导出

在结果页 / 任务页点击"导出"，选择 PDF 或 Excel：

- **PDF**：含等级、量化指标、缺陷、溯源二维码，适合打印归档。
- **Excel**：含结果明细表，适合数据分析。

### 3.7 移动端 H5 使用

手机浏览器打开系统地址，登录后在检测页点击"拍照上传"，调用摄像头拍摄柑橘即可检测，结果查看方式与 PC 一致。

## 4. 管理员操作指南

管理员登录后，导航栏出现"系统管理"入口。

### 4.1 分级标准管理

1. 进入"标准管理"（admin/Standard）。
2. 查看标准列表（砂糖橘 / 贡柑等）。
3. 编辑阈值：最小果径、着色率阈值 1（一级）/ 2（二级）、缺陷占比阈值 1 / 2。
4. 保存并"激活"某标准（同一时刻仅一个生效）。
5. 后续检测按激活标准分级。

> 默认砂糖橘标准：最小果径 45mm、着色率 85%/70%、缺陷占比 2%/5%。

### 4.2 价格管理

进入"价格管理"（admin/Price），维护各等级（grade1/grade2/out）的参考价与单位。默认：一级 8.50、二级 5.00、等外 2.00 元/斤。

### 4.3 模型管理

进入"模型管理"（admin/Model），查看模型版本列表（版本号、算法、mAP/P/R），选择某版本"激活"，算法服务后续推理加载该模型。

### 4.4 日志查询

进入"日志管理"（admin/Log），按操作类型（login/upload/detect/export）与时间分页查询操作记录。

### 4.5 统计看板

仪表盘展示检测总量、各等级占比饼图、缺陷类型分布柱状图（ECharts 渲染）。

## 6. 常见问题（FAQ）

参考资料汇总见 [docs/REFERENCES.md](../REFERENCES.md)。

| 问题 | 解决方法 |
|---|---|
| **YOLO 训练显存不足** | 减小 batch size、用 YOLOv8n、降低输入分辨率（如 640→416），或用 Colab/AI Studio 免费 GPU |
| **MySQL 连接失败** | 检查 `DATABASE_URL`、端口 3306、用户名密码；Docker 下确认 mysql 容器 healthy |
| **前端跨域（CORS）** | 后端 FastAPI 配置 CORSMiddleware 允许前端域名；或经 nginx 同源反代 `/api` |
| **Docker 拉取镜像慢** | 配置国内镜像加速器（阿里云 / 腾讯云镜像源） |
| **演示视频超 50M** | 用 HandBrake / ffmpeg 压缩，降码率 / 分辨率，控制时长 |
| **检测很慢（>3s）** | 确认走 CPU 推理且用 YOLOv8n；首次推理含模型加载，第二次起变快 |
| **上传被拒** | 确认格式为 JPG/PNG 且 ≤10MB |
| **算法服务连不上** | 确认 :8001 已启动；后端会自动降级为传统 CV 分级（无缺陷框） |

- 首次登录请修改默认密码。
