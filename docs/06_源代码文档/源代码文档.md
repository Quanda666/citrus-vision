### 3.2 分层职责

| 层 | 目录 | 职责 |
|---|---|---|
| 路由层 | `api/v1` | 解析请求、调 Service、封装统一返回 |
| 业务层 | `services` | 业务编排、事务、调算法服务 |
| 数据访问层 | `crud` | 封装 ORM 增删改查 |
| 模型层 | `models` | SQLAlchemy 实体（对应 10 张表） |
| 数据传输 | `schemas` | Pydantic 校验与序列化 |
| 核心 | `core` | 配置、JWT、bcrypt、依赖注入 |

### 3.3 关键类 / 函数清单

| 文件 | 类 / 函数 | 职责 |
|---|---|---|
| core/security.py | `hash_password` / `verify_password` | bcrypt 哈希与校验 |
| core/security.py | `create_token` / `decode_token` | JWT 签发与解析 |
| core/deps.py | `get_current_user` / `require_admin` | 鉴权依赖 |
| services/auth_service.py | `AuthService` | 注册 / 登录 / 验证 |
| services/detection_service.py | `DetectionService` | 创建任务、编排检测、兜底 |
| services/detection_service.py | `AlgorithmClient.infer` | 调算法服务 `/infer` |
| services/sample_service.py | `SampleService` | 上传 / 检索 / 删除 |
| services/report_service.py | `ReportService` | PDF / Excel 导出 |
| services/trace_service.py | `TraceService` | 溯源 + 二维码 |
| utils/response.py | `success` / `error` | 统一返回包 `{code,msg,data}` |

## 4. 算法服务代码组织（algorithm-service/）

### 4.1 目录

```

├── server.py                  # FastAPI 入口，提供 POST /infer
├── requirements.txt

│   ├── segment.py             # 果实分割/ROI
│   ├── detect.py              # YOLOv8 缺陷检测
│   ├── features.py            # 特征计算

│   └── evaluate.py            # 评估（mAP/PR/混淆矩阵）
```

### 4.2 关键模块函数

| 文件 | 函数 / 类 | 职责 | 关键参数 |
|---|---|---|---|
| pipeline/preprocess.py | `Preprocessor.process` | 缩放 / 去噪 / 白平衡 / CLAHE | resize 尺寸、CLAHE clipLimit |
| pipeline/segment.py | `Segmenter.segment` | HSV 阈值 + 形态学分割果实 | HSV 橙色区间、核大小 |
| pipeline/segment.py | `Segmenter.bounding_circle` | 求外接圆（果径） | — |
| pipeline/detect.py | `Detector.detect` | YOLOv8 推理 | CONF=0.25、IOU=0.45 |
| pipeline/features.py | `diameter / color_ratio / defect_ratio` | 三项指标计算 | 像素→mm 换算 |
| pipeline/grade.py | `Grader.grade` | 规则引擎判级 | 标准阈值（85/70、2/5、45mm） |
| server.py | `infer` | 编排流水线，返回 InferResult | — |
| train/evaluate.py | `evaluate` | 计算 mAP/P/R/混淆矩阵 | — |

## 6. 编码规范

### 6.1 通用

- **命名**：后端 Python `snake_case`（类 `PascalCase`）；前端 TS `camelCase`（组件 `PascalCase`）；数据库字段 `snake_case`。
- **注释**：业务逻辑、算法关键步骤、非显然分支必须有中文注释；公共函数写 docstring（入参 / 返回 / 异常）。
- **统一返回**：后端所有接口经 `utils/response.py` 封装 `{code, msg, data}`，不裸返回。
- **错误码**：遵循概要设计错误码表（0/400/401/403/404/500）。

### 6.2 提交规范

Git 提交信息用类型前缀：`feat`（功能）、`fix`（修复）、`docs`（文档）、`refactor`、`test`、`chore`。分支：`feat/*`，由组长合并到 `main`。

### 6.3 安全要点

- 密码仅存 bcrypt 哈希，禁止明文。
- JWT 密钥经环境变量注入，不入库不进版本库。
- 上传文件校验类型与大小，限定存储目录。
- 管理接口经 `require_admin` 依赖校验。

## 7. 构建与运行

| 服务 | 安装 | 启动 |
|---|---|---|
| 后端 | `pip install -r backend/requirements.txt` | `uvicorn main:app --port 8000` |

| 前端 | `npm install`（frontend/） | `npm run dev`（:5173）/ `npm run build` |
| 数据库 | — | `mysql < database/schema.sql && mysql < database/seed.sql` |
| 全栈 | — | `docker-compose up -d` |
