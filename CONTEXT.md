# CONTEXT.md

## 项目领域上下文

### 项目概述

CitrusVision 是一个面向肇庆本地柑橘产业的机器视觉品质检测系统，用于解决传统人工分级的主观性强、标准不一、效率低、缺陷漏检等问题。

### 领域术语（Ubiquitous Language）

#### 业务领域

- **品质分级（Quality Grading）**：根据果径、色泽、缺陷等指标将柑橘分为一级（精品）、二级（合格）、等外品三个等级
- **缺陷检测（Defect Detection）**：识别并定位柑橘表面的病斑、裂纹、碰伤、畸形等缺陷
- **溯源信息（Traceability）**：包含产地、批次、检测时间、检测人等可追溯数据
- **着色率（Color Ratio）**：果实表面橙色区域占比，表征成熟度和色泽品质
- **果径（Diameter）**：果实直径，品质分级的重要指标之一
- **缺陷占比（Defect Ratio）**：缺陷区域面积占果面总面积的百分比

#### 技术领域

- **ROI（Region of Interest）**：感兴趣区域，指从图像中分割出的果实区域
- **预处理（Preprocessing）**：包括去噪、白平衡、归一化等图像增强操作
- **果实分割（Fruit Segmentation）**：将果实从背景中分离的过程
- **置信度（Confidence）**：模型对检测结果的确信程度，范围 0-1
- **mAP（Mean Average Precision）**：目标检测模型的主要评估指标
- **双引擎方案**：传统 CV（可解释分级）+ 深度学习（缺陷检测）的混合架构

#### 用户角色

- **普通用户（User）**：产地分拣员、合作社质检员，执行日常检测任务
- **管理员（Admin）**：技术员、管理人员，维护系统配置、标准、模型

### 业务规则与约束

#### 分级标准

| 等级 | 判定规则 |
|---|---|
| 一级（精品）| 果径达标 且 着色率 ≥ 85% 且 缺陷占比 < 2% 且 无裂纹/腐烂 |
| 二级（合格）| 着色率 ≥ 70% 且 缺陷占比 < 5% 且 无裂纹/严重畸形 |
| 等外品 | 着色率 < 70% 或 缺陷占比 ≥ 5% 或 存在裂纹/严重畸形/腐烂 |

*注：阈值可由管理员在"标准管理"模块配置*

#### 缺陷类别

- **病斑（Black Spot）**：黑褐色斑点，病原菌感染
- **裂纹（Crack）**：果皮开裂，影响储存和销售
- **碰伤（Bruise）**：机械损伤导致的软化或变色
- **畸形（Deformity）**：形状不规则、发育不良

#### 性能约束

- 单张检测响应时间（冷启动）：≤ 5 秒（CPU，含模型加载）/ ≤ 2 秒（GPU）
- 单张检测响应时间（后续）：≤ 3 秒（CPU）/ ≤ 1 秒（GPU）
- 批量检测（10 张）：≤ 30 秒（后续）
- 图片格式：JPG/PNG，单张 ≤ 10MB
- 并发支持：演示级 ≥ 5 并发

### 核心业务流程

```
用户上传图片 → 系统预处理 → 果实分割提取 ROI → 
  ├─ YOLOv8 缺陷检测（框选+分类）
  ├─ 特征计算（果径/着色率/缺陷占比）
  └─ 规则引擎分级 → 结果可视化 + 溯源登记 + 报表导出
```

### 架构约束

- **四层分离**：前端、后端 API、算法服务、数据库各自独立
- **算法解耦**：算法服务通过内部 HTTP 接口调用，便于模型替换
- **可解释性优先**：分级必须附带量化指标（果径/着色率/缺陷占比），不能黑盒
- **传统兜底**：深度学习效果不佳时，传统 CV 分级链路必须独立可用

### 关键非功能需求

- **可用性**：界面中文、操作 ≤ 3 步、移动端友好
- **可维护性**：分级标准可配置、模型可热切换
- **可演示性**：缺陷框选醒目、等级一目了然、支持实拍演示
- **可追溯性**：每次检测记录入库，可查询、可导出

### 技术选型依据

- **FastAPI**：自带 Swagger 文档，直接服务接口设计交付
- **YOLOv8n**：CPU 可推理，适合无 GPU 环境
- **Vue 3 + Element Plus**：学习曲线平、组件丰富、出活快
- **MySQL**：符合课设"数据库设计"要求，便于画 E-R 图
- **双引擎混合**：传统 CV 可解释 + DL 高精度，双保险

### 数据约束

- 数据集规模：演示级 300~600 张（每类缺陷 80~150 张）
- 数据来源：公开数据集 + 网络采集 + 自建小样本（肇庆本地实拍）
- 划分比例：训练:验证:测试 = 7:2:1
- 标注格式：YOLO txt 格式（缺陷框）+ 图像级类别标签（分级）

### 项目目标（课设导向）

**核心策略**：文档与系统完整性吃满（分析 24 分 + 设计 24 分 = 48 分大头），算法只做到"能演示、有兜底"（实现仅 14 分）。

- ✅ 需求分析：用例图、DFD 1 层、数据字典详尽
- ✅ 概要设计：软件结构图/包图、E-R 图完整
- ✅ 详细设计：类图、时序图、活动图、状态图齐备
- ✅ 系统实现：MVP 闭环可用，算法达演示级
- ✅ 报告撰写：按模板规范、排版达标、按时提交
