## 📥 第二步：数据采集（Day 1-2，张嘉豪负责）

### 2.1 下载公开数据集

```bash
# Kaggle Citrus Disease Dataset
cd data/raw/public
# 方法1：在 Kaggle 网页手动下载 zip（推荐）
# 方法2：用 kaggle CLI（需先配置 API key）
# kaggle datasets download -d dtlsr/citrus-disease

# 解压并重命名
unzip citrus-disease.zip
# 重命名为 public_001.jpg, public_002.jpg...
```

### 2.2 网络采集图片

```bash
cd data/raw/web

# 方法1：手动下载（推荐，安全）
# 淘宝搜索"砂糖橘 缺陷"，右键保存图片

# 方法2：Bing 图片下载器（合法）
pip install bing-image-downloader
python -c "from bing_image_downloader import downloader; downloader.download('citrus defects', limit=100, output_dir='.')"
```

**注意**：过滤掉水印、低分辨率（<500px）、无关图片。

### 2.3 自建实拍（Day 6-7 周末，全组）

**拍摄清单**：
- [ ] 正常果：20 张
- [ ] 病斑果：30 张
- [ ] 裂纹果：20 张
- [ ] 碰伤果：20 张
- [ ] 畸形果：10 张

**拍摄规范**：
- 纯色背景（白布/黑布）
- 放硬币参照（1元，直径25mm）
- 固定距离（约30cm）
- 自然光，避免强反光

---

## 🏷️ 第三步：标注（Day 3-5，2人并行）

### 3.1 Roboflow 标注流程

1. 上传图片到项目
2. 点击图片 → Annotate
3. 按 `B` 键绘制矩形框
4. 框选缺陷，选择类别：
   - `black_spot`（病斑）
   - `crack`（裂纹）
   - `bruise`（碰伤）
   - `deformity`（畸形）
5. 保存（自动保存）

### 3.2 LabelImg 标注流程（备用）

```bash
labelImg data/raw/labeled data/classes.txt
```

- `W` 键：绘制框
- `A/D` 键：切换上一张/下一张
- 保存格式：YOLO

## 🚨 常见问题

### Q1：Roboflow 上传图片很慢怎么办？
**A**：分批上传，每次 50 张；或切换到 LabelImg 本地标注。

### Q2：某类缺陷样本找不够怎么办？
**A**：优先保证 3 类（病斑/裂纹/碰伤）各 ≥80 个，畸形可以少一些（≥50）。

### Q3：Day 5 才完成 200 张，进度延迟怎么办？
**A**：缩减目标到 300 张，优先保证质量；或加班周末补。

### Q4：标注时不确定某个斑点是不是病斑？
**A**：遵循"保守原则"：不确定的不标；复核时再裁定。
