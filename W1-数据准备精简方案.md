## 🏷️ 第二步：快速复核标注（Day 2-3，12-18 小时）

### 2.1 为什么要复核

公开数据集的标注可能存在问题：
- 类别定义不一致（他们的 "disease" 可能包含我们的 "病斑+裂纹"）
- 标注框不准确（过大/过小/漏标）
- 格式需要转换（Pascal VOC → YOLO）

### 2.2 复核分工

| 人员 | 任务 | 工作量 |
|---|---|---|
| 张嘉豪 | 复核 Kaggle 数据集（~100 张） | 6-9 小时 |
| 黄浩然 | 复核 Roboflow 数据集（~100 张） | 6-9 小时 |

**并行工作**：Day 2 下午 + Day 3 全天

### 2.3 复核标准

**快速复核流程**（每张约 3-5 分钟）：

1. 用 LabelImg 打开图片和标注
2. 快速检查：
   - 是否有缺陷框（无框 → 补标）
   - 框是否准确（偏差大 → 调整）
   - 类别是否正确（对照我们的 4 类）
3. 保存（YOLO 格式）

**我们的 4 类映射**：

| 我们的类别 | 公开数据集可能的类别名 |
|---|---|
| black_spot（病斑） | disease, black_spot, canker |
| crack（裂纹） | crack, split |
| bruise（碰伤） | bruise, damage, wound |
| deformity（畸形） | deformed, irregular |

**复核工具**：
```bash
pip install labelImg
labelImg data/labeled data/classes.txt
```

**classes.txt**：
```
black_spot
crack
bruise
deformity
```

## 🚨 常见问题

### Q1：200 张够吗？
**A**：够。YOLOv8 从预训练模型微调，200 张能收敛。参考：
- YOLOv8 官方最小示例：128 张
- 课设不是论文，能演示即可

### Q2：公开数据集的标注格式不是 YOLO 怎么办？
**A**：Roboflow 导出时选 YOLOv8 格式；Kaggle 若是 Pascal VOC（xml），用转换脚本：
```python
# scripts/voc_to_yolo.py（如果需要，我可以提供）
```

### Q3：某类样本只有 20 个怎么办？
**A**：可接受。数据增强（旋转/翻转）会扩充 3-5 倍，实际训练样本 60-100 个。

### Q4：答辩时被问"为什么只有 200 张"？
**A**：诚实回答：
> "受限于时间和资源，当前数据集为 200 张公开数据。我们的系统架构已支持数据扩展，后续可通过增加自建标注数据提升模型精度。课设重点是验证完整系统的可行性。"
