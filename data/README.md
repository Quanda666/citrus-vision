# 数据集目录

本目录存放 CitrusVision 项目的数据集。

## 目录结构

```
data/
├── raw/                     # 原始采集数据（未处理）
├── images/                  # 图像数据（YOLO 格式）
│   ├── train/              # 训练集图像
│   ├── val/                # 验证集图像
│   └── test/               # 测试集图像
├── labels/                  # 标注数据（YOLO txt 格式）
│   ├── train/              # 训练集标注
│   ├── val/                # 验证集标注
│   └── test/               # 测试集标注
├── grade/                   # 分级数据（图像级分类）
│   ├── grade1/             # 一级（精品）
│   ├── grade2/             # 二级（合格）
│   └── grade_out/          # 等外品
├── annotations/             # 标注源文件（LabelImg/Roboflow 导出）
└── README.md               # 本文件
```

## 数据获取策略（三源混合）

### 1. 公开数据集（打底）

- Kaggle 柑橘/水果缺陷数据集
- 百度 AI Studio 相关数据集
- 天池农产品质检数据集
- 扩展对照：苹果、番茄数据集

### 2. 网络采集（筛选）

- 搜索引擎图像搜索
- 电商平台商品图
- 人工剔除水印、重复、无关图像

### 3. 自建小样本（肇庆本地实拍）

- **拍摄要求**：
  - 纯色背景（白色/黑色）
  - 固定距离（30-40cm）
  - 放硬币作参照物（用于果径换算）
  - 光照均匀、无强反光
  - 覆盖不同等级和缺陷类型
  
- **数量目标**：
  - 每个等级 ≥ 50 张
  - 每种缺陷 ≥ 80 张
  - 总计 300~600 张（演示级规模）

## 标注规范

### 缺陷标注（YOLO 格式）

使用 LabelImg 或 Roboflow 标注，导出 YOLO txt 格式：

```
<class_id> <x_center> <y_center> <width> <height>
```

- 所有坐标归一化到 [0, 1]
- class_id 映射：0-病斑, 1-裂纹, 2-碰伤, 3-畸形

**标注约定**：
- 最小框尺寸：≥ 10×10 像素
- 模糊缺陷：置信度不足时跳过
- 重叠缺陷：按主要类型标注

### 分级标注（图像级）

按等级分文件夹存放：
- `grade/grade1/` - 一级（精品）
- `grade/grade2/` - 二级（合格）
- `grade/grade_out/` - 等外品

## 数据划分

**比例**：训练:验证:测试 = **7:2:1**

**分层抽样**：按等级/缺陷类别分层，保证各集分布一致

## 数据增强

使用 YOLOv8 内置增强 + 自定义增强：

- 旋转：±15°
- 翻转：水平/垂直
- 亮度/对比度：±20%
- 加噪：高斯噪声
- Mosaic：YOLOv8 自动

**增强倍数**：3~5 倍

## 数据集元信息

创建 `data_card.md` 记录：
- 数据来源
- 采集时间
- 标注人员
- 类别分布
- 质量检查结果
