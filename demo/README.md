## 演示样本要求

### 样本来源

1. **肇庆本地实拍**（强调地方特色）
   - 四会砂糖橘 5 张
   - 德庆贡柑 5 张
   - 纯色背景（白色泡沫板/黑色绒布）

   - 自然光拍摄

2. **公开数据集精选**
   - 选取典型缺陷样本

### 样本分类

| 等级 | 数量 | 特点 | 用途 |
|---|---|---|---|
| 一级（精品） | 5 张 | 色泽均匀、无瑕疵、果型饱满 | 展示最佳检测效果 |
| 二级（合格） | 5 张 | 轻微色差、小缺陷 | 展示分级准确性 |
| 等外品 | 5 张 | 明显裂纹/病斑/畸形 | 展示缺陷检测能力 |

**关键要求**：
- 缺陷必须肉眼可见（病斑/裂纹清晰）
- 避免曝光过度或过暗
- 每张图片独立可演示，也可组合批量

## 视频制作要求

### 技术要求

- 分辨率：1920×1080 (1080p)
- 帧率：30 fps
- 编码：H.264
- 音频：AAC, 128 kbps
- **文件大小：≤ 50M**（红线）

### 压缩策略

如果原视频超过 50M：

```bash
# 使用 ffmpeg 压缩
ffmpeg -i input.mp4 -vcodec h264 -crf 28 -preset slow -acodec aac -b:a 96k output.mp4

# 参数说明：
# -crf 28: 质量因子（18-28 可接受，数值越大压缩越强）
# -preset slow: 编码速度（slow 压缩率更高）
# -b:a 96k: 音频码率降低
```

### 配音/字幕

- 配音：普通话，语速适中，避免口头禅
- 字幕：关键操作步骤添加字幕说明
- BGM：轻音乐，音量 -20dB（不抢戏）

## 答辩现场演示流程

### 准备工作

1. **双保险**：
   - 主方案：本地离线演示（localhost）
   - 备份：预录视频（U 盘 + 云盘）

2. **环境检查**：
   - 提前启动所有服务（backend/algorithm/frontend）
   - 准备演示图片到桌面快捷文件夹
   - 测试完整流程 ≥2 次

3. **设备准备**：
   - 笔记本电脑 + HDMI/VGA 转接线
   - 手机（演示 H5 拍照上传）
   - 移动热点（以防网络故障）

### 演示步骤（5 分钟）

1. **开场白**（30 秒）
   - 自我介绍 + 项目背景
   - 核心亮点：双引擎方案 + 可解释分级

2. **现场实拍**（1 分钟）⭐亮点
   - 拿出准备好的砂糖橘
   - 手机扫码进 H5，拍照上传
   - 秒级返回结果，展示缺陷红框 + 等级徽章

3. **批量处理**（1 分钟）
   - 批量上传 10 张
   - 展示统计图表

4. **可解释性**（1 分钟）
   - 点开某个结果详情
   - 强调量化指标：果径/着色率/缺陷占比
   - 说明分级依据（与 PPT 中标准对应）

5. **工程完整性**（1 分钟）
   - 快速翻阅文档（UML/E-R/Swagger）
   - 展示 Docker Compose 一键部署

6. **收尾**（30 秒）
   - 回顾创新点
   - 感谢评委

### 应急预案

| 故障 | 应对 |
|---|---|
| 服务启动失败 | 立即切换到预录视频 |
| 网络/API 超时 | 展示离线截图 + 口述流程 |
| 手机 H5 无法访问 | PC 端上传演示代替 |
| 投影设备不兼容 | 在笔记本屏幕上演示 |
